Q 3.Mention some important git commands.

Ans 3.
git-command use
git clone Git clone is a command for downloading existing source code from a remote repository (like Github, for example). In other words, Git clone basically makes an identical copy of the latest version of a project in a repository and saves it to your computer.

Syntax: clone repository-link
git status The Git status command gives us all the necessary information about the current branch. like
    Whether the current branch is up to date
    Whether there is anything to commit, push or pull
    Whether there are files created, modified or deleted

Syntax: git status
git add When we create, modify or delete a file, these changes will happen in our local and won't be included in the next commit (unless we change the configurations).
We need to use the git add command to include the changes of a file(s) into our next commt.
    To add everything at once: use -A (inplace of file_name)

Syntax: git add file_name.(with extension)
git commit Once we reach a certain point in development, we want to save our changes (maybe after a specific task or issue).
Git commit is like setting a checkpoint in the development process which you can go back to later if needed.
We also need to write a short message to explain what we have developed or changed in the source code.

Syntax: git commit -m "commit message"
Git push After committing your changes, the next thing you want to do is send your changes to the remote server. Git push uploads your commits to the remote repository.
Important: Git push only uploads changes that are committed.
We also need to write a short message to explain what we have developed or changed in the source code.

Syntax: git push -u origin branch_name
Git branch By using branches, several developers are able to work in parallel on the same project simultaneously.
We can use the git branch command for creating, listing and deleting branches.

Creating a new branch: git branch branch_name

To push the new branch into the remote repository: git push -u branch_name

Viewing branches: git branch or git branch --list

Deleting a branch: git branch -d branch_name
Git checkout We use git checkout mostly for switching from one branch to another. We can also use it for checking out files and commits.
Syntax: git checkout name-of-your-branch
Syntax: git checkout commit_id(to which commit you want to go)

The below command creates a new branch in your local (-b stands for branch) and checks the branch out to new right after it has been created.

Syntax: git checkout -b name-of-your-branch
Git merge When you've completed development in your branch and everything works fine, the final step is merging the branch with the parent branch (dev or master). This is done with the git merge command.
It's important to remember that you first need to be on the specific branch that you want to merge with your feature branch.
For example, when you want to merge your feature branch into the dev branch: First you should switch to the dev branch:

Syntax: git merge branch_name
Git log It shows all the previous commits.

Syntax: git log
